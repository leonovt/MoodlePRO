MoodlePRO — Plan & Build Status

Live Hebrew/English transcription, captions, and downloads for BGU Moodle lecture videos. Self-hosted, three-tier architecture. Snapshot as of 2026-06-19.

Why this architecture

Originally scoped as a BYOK/Base44 hackathon project (Groq Whisper API, no GPU, Chrome Web Store BYOK distribution). Pivoted to a fully self-hosted pipeline because the user owns an RTX 4090 and wants to own the whole stack end-to-end instead of paying per-call cloud ASR or depending on Base44. The BGU-specific DOM detection facts from the earlier plan (video.js player, direct CloudFront MP4, numeric local/video_directory video IDs, no native captions) carried over unchanged.

StepWhat happens
1Content script detects the BGU video.js MP4 player, grabs the direct video URL + numeric Moodle video id.
2POST /jobs → server downloads the video, extracts 16kHz mono audio with ffmpeg, hashes it (SHA-256).
3Hash hit in Postgres → cached transcript returned immediately. Miss → job queued in Redis, job_id returned.
4Client opens WS /ws/jobs/{job_id}.
5GPU worker pops the job, pulls audio over HTTP (GET /internal/audio/{id}, bearer token), transcribes, publishes segments to Redis pub/sub as produced — server relays them to the websocket live.
6Worker posts the finished transcript (POST /internal/jobs/{id}/complete) — server persists it keyed by audio hash (reused forever), pushes a final completed event.
7"Download Transcript" hits GET /jobs/{id}/txt and /srt.

The worker only makes outbound connections (to Redis and the server's HTTP API) — it can run on a home GPU machine behind NAT with no port-forwarding.

Component status

🧩 extension/ Built

Manifest V3 Chrome extension. BGU-specific video.js/MP4 detection, sidebar transcript panel, caption overlay, REST + WebSocket client, download button. Bundled with esbuild (content scripts can't use ES import directly).

Tested: Vitest + jsdom — 4 test files (api-client, detect-player, inject, segment-sync), 15 tests.

⚙️ server/ Built

FastAPI orchestration: yt-dlp/HTTP download → ffmpeg extraction → SHA-256 dedup → Postgres transcript cache → Redis job queue + pub/sub → WebSocket relay.

Tested: pytest against SQLite + fakeredis (no real Postgres/Redis needed) — test_jobs.py, test_ws.py, 6 tests.

🖥️ gpu_worker/ Stubbed

Pops jobs from Redis, fetches audio via authenticated HTTP GET, transcribes, posts completion back. Ships with FAKE_TRANSCRIBE=true (canned Hebrew segments).

Tested: test_transcriber.py (stub logic) + test_integration.py, which runs the real FastAPI server in-process to verify the full queue/HTTP contract.

Not tested: real transcription — no GPU/model wired up yet.

🧠 Summarization Not started

server/app/services/summarizer.py defines a SummarizerProvider ABC as an extension point. The default implementation raises NotImplementedError.

Tested: nothing — no provider implemented yet, no LLM wired in.
